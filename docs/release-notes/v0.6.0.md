# Spec Lifecycle Manager v0.6.0

## Highlights

- Adds a unified singular `slm spec` interface for listing specs and opening a
  spec's tasks, next work, or requirements.
- Shows phase progress and phase state in the spec inventory when task-backed
  phases exist, using the same normalized task states as the task views.
- Restores Claude and Codex MCP tool discovery by requiring every declared MCP
  output schema to have a top-level JSON object type.

## Added

- Added `slm spec [all|open|closed]`, defaulting to `open`.
- Added `slm spec <number> [tasks|next|requirements]`, defaulting to `tasks`.
- Added `PHASES` and `PHASE STATE` columns to the active-spec inventory.
- Added `phases_complete`, `phases_total`, `current_phase`, and `phase_state` to
  the versioned JSON inventory contract. Specs without explicit task-backed
  phases report null phase values rather than inferred progress.

## Changed

- Phase state is derived from normalized task states, including attention,
  review-needed, in-progress, partial, pending, follow-up, no-op, and complete
  states.
- Existing plural commands remain available for compatibility while the
  singular `slm spec` command provides the shorter navigation path.
- Package, Codex plugin, Claude plugin, pinned marketplace, and build metadata
  now identify version `0.6.0`.

## Fixed

- Corrected MCP `outputSchema` declarations whose top-level type was not
  `object`. Claude and Codex can now validate and fetch the complete tool list
  instead of rejecting it during plugin reconnection.
- Kept source, Codex, and Claude runtime bundles synchronized for the new CLI,
  phase projection, and schema behavior.

## Documentation

- Updated the README, lifecycle design, runtime reference, and MCP install
  guide for the singular command routes, defaults, phase semantics, and
  `v0.6.0` package metadata.
- Reconciled Spec 039 requirements, design, traceability, tasks, quickstart,
  change impact, canonical context, and verification evidence.

## Validation

- The final `npm run validate` release-candidate run passed with 360 Python
  tests and 33 Node tests, plus lifecycle scan, archive-index and prompt
  validation, package contract, source/bundle parity, installed-tarball smoke,
  npm package dry-run, and whitespace checks.
- Focused public-view and CLI coverage passed 38 tests, including phase state
  precedence, phase-less specs, singular command defaults, and JSON output.

## Known Issues

- Existing Codex and Claude sessions must install or update to `v0.6.0` and be
  restarted or reloaded before they use the corrected bundled MCP server.
- The package is distributed through the GitHub release tarball; it is not
  currently published in the npm registry.
