# Spec Lifecycle Manager v0.4.0

## Highlights

- Adds bounded, MCP-first phase and task-start workflows that preserve
  authoritative blockers while keeping detailed evidence available through
  deterministic expansion.
- Makes spec creation and closure safer with repository-owned ID allocation,
  preview-first creation plans, compact closure manifests, and guarded cleanup
  records.
- Reduces routine agent context and hook noise: the source skill entrypoint is
  69.2% smaller, ordinary write hooks stay narrow, and stale cache hook paths
  fail quietly.
- Publishes the package as `@auriora/ai-spec-lifecycle` and keeps Codex and
  Claude plugin payloads synchronized at version 0.4.0.

## Added

- Added `phase_gate_check`, a read-only eight-phase decision facade with
  bounded compact/full/section responses, blocker-first ordering, content
  fingerprints, and MCP/CLI invocation provenance.
- Added archive-aware `spec_id_inventory` and preview-only
  `spec_creation_plan` surfaces so agents do not select spec numbers manually
  or mistake a proposal for an atomic reservation.
- Added the declarative `implementation-start` prompt, which composes active
  preflight, task context, traceability, readiness, and validation planning for
  one selected task without mutation.
- Added packaged build identity and lifecycle capability reporting. Optional
  MCP client identity is retained only in process memory and does not influence
  lifecycle readiness or next actions.
- Added repository-local development discovery through `.agents/skills`,
  `.codex/config.toml`, `.codex/hooks.json`, and
  `scripts/codex-spec-lifecycle-dev.sh`.

## Changed

- Lifecycle next actions now route through validation planning, evidence
  quality, and durable promotion before closure. MCP tools are primary;
  retained CLI commands are explicitly labelled for CI, hooks, debugging, or
  recovery.
- The source `SKILL.md` entrypoint was reduced from 53,427 to 16,452 bytes while
  preserving eight mandatory rule categories through direct named references.
- Closure MCP responses now omit replacement bodies, expose targeted
  expansions, enforce plan fingerprints, and support independently guarded,
  idempotent record and package actions.
- Archived-package and closure scans now honor repository ignores and exclude
  caches, VCS metadata, databases, WAL/SHM files, and binary payloads.
- User-wide installation is package-owned. A Git checkout is refused when it
  targets the default user Codex home; checkout installer tests require
  explicit isolated roots.

## Fixed

- Fixed MCP default repository binding so the packaged launcher preserves the
  client working directory unless an explicit repository root is supplied.
- Fixed duplicate JSON in MCP text envelopes while retaining authoritative
  `structuredContent`.
- Fixed ordinary spec, task, template, and verification hooks so they neither
  execute nor recommend full-package lint. Explicit resume, closure, and direct
  validation boundaries retain full checks.
- Fixed repeated hook noise with debounce and quiet-success behavior. Packaged
  Codex and Claude hook launchers now exit successfully without feedback when
  a running session references a removed versioned cache script.
- Tightened evidence-quality classification and prevented runnable tasks from
  being mistaken for phase completion.

## Documentation

- Reworked the README and runtime/install references around MCP-first use,
  package-backed installation, repository-local development, and explicit
  recovery boundaries.
- Added durable guidance for phase gates, provisional spec allocation, compact
  response contracts, closure records, client capability observation, and
  qualified dogfood evidence.
- Closed and indexed Specs 033, 035, 036, 037, and 038 after promoting their
  accepted behavior to durable documentation.

## Validation

- `npm run validate` passed on 2026-07-19 with 321 Python tests and 26 Node
  tests.
- Lifecycle scan, archive-index validation, 11 prompt definitions, package
  contract, source/bundle sync guard, npm pack dry-run, and whitespace checks
  passed.
- Cross-platform installer, MCP-launch, source-discovery, advisory-hook, and
  missing-cache smoke coverage is included for Windows, macOS, and Linux
  workflows.

## Known Issues

- Existing Codex or Claude sessions must be restarted or reloaded to pick up a
  newly installed plugin version.
- The repository development launcher disables packaged plugins for that
  process because Codex does not currently provide project-scoped enablement
  for one installed plugin.
- External Chat Analyser operation distributions remain unavailable where
  capability attribution and native-to-normalized reconciliation are
  incomplete; this does not affect lifecycle runtime availability.
