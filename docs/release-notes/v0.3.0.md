# Spec Lifecycle Manager v0.3.0

## Highlights

- Adds a package-backed release process with generated release-note evidence,
  reviewed release notes, GitHub release publication, and guarded npm publish
  support.
- Expands the lifecycle runtime around MCP-first usage, closure helpers,
  canonical-context diagnostics, requirement priorities, and agent task context.
- Aligns Codex and Claude plugin payloads with the source skill, prompts,
  runtime helpers, templates, and package metadata.

## Added

- Added the repo-local `slc` developer CLI and package workflow commands for
  validation, packaging, local plugin installation, sync checks, release
  preflight, release notes, tagging, and GitHub release creation.
- Added release-note generation support through `slc release notes`, including
  JSON evidence, a draft note, and agent refinement instructions.
- Added MCP-first lifecycle runtime surfaces for migrated tools, including
  task context, traceability lookup, closure helper behavior, prompt
  validation, archive-index validation, and package contract checks.
- Added requirement priority support for accepted requirements, including
  parsing, coverage semantics, authoring guidance, agent context propagation,
  and validation behavior.

## Changed

- Shifted lifecycle guidance toward MCP tools as the primary agent-facing
  surface, while keeping `spec_runtime.py` commands for CI, recovery, and
  runtime debugging.
- Refined canonical-context guidance so missing context is advisory unless it
  affects current implementation authority.
- Strengthened closure handling with preview-first cleanup, closure-hash
  resolution, durable closure records, and active-package cleanup boundaries.
- Standardized install and package guidance around the release tarball/npm
  package path, with Codex and Claude plugin copies kept in sync with source.

## Fixed

- Corrected CI and cross-platform test dependency behavior around the package
  and runtime validation paths.
- Fixed MCP workspace binding and path assertions so installed and source
  runtime checks resolve repository context consistently.
- Addressed review findings in requirement-priority behavior and documentation.

## Documentation

- Updated the front-door README, agent instructions, runtime reference, MCP
  install reference, design docs, backlog, roadmap, and closure history to
  reflect package-backed installation and MCP-first lifecycle operation.
- Added and closed durable spec history for the completed lifecycle slices in
  this release, including dev CLI workflow tools, npm release workflow, guided
  documentation wizard, MCP-first runtime migration, closure helper, canonical
  context warning behavior, and requirement priority labels.
- Added the `release-notes` skill to source and bundled plugin copies so future
  releases can be prepared from Git evidence.

## Validation

- `npm run validate` passed on 2026-07-06 after the release tooling changes.
- Release evidence was generated with `slc release notes` for the range
  `v0.2.1..HEAD`.

## Known Issues

- GitHub PR metadata enrichment is not included in the generated release-note
  evidence; the current workflow uses local Git history and validation notes.
- Release notes still require human or agent review after evidence generation;
  the draft output is intentionally not treated as publishable prose.
