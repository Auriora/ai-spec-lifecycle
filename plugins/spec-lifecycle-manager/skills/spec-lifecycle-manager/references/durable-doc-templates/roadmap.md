# Roadmap

Use this template for sequenced work, milestones, adoption stages, or
multi-spec dependencies. A roadmap is a planning view, not a product changelog
and not a replacement for active implementation specs.

## Status Values

| Status | Meaning |
|--------|---------|
| proposed | Candidate roadmap item without planning commitment. |
| planned | Accepted for a future horizon. |
| active | Currently being delivered or actively coordinated. |
| complete | Exit criteria met and evidence linked. |
| deferred | Postponed with a review trigger or dependency. |
| dropped | No longer planned; reason recorded in notes or decision history. |

## Routing Rules

- Use roadmap for sequencing, milestones, adoption stages, or dependencies that
  span multiple specs or backlog items.
- Link to backlog for work that still needs clarification before it can be
  scheduled.
- Link to active specs or issues for implementation-ready work.
- Link to closure logs when completed specs produce follow-up roadmap work.
- Do not use the roadmap as a product changelog; record completed delivery
  evidence and keep user-facing release history in the repository's changelog
  if one exists.
